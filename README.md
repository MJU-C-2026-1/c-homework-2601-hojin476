# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 오운완 (오늘 운동 완료) 기록기

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자의 신체 정보와 웨이트 트레이닝 기록을 입력받아 운동 성과를 분석하고 목표 달성률을 관리하는 헬스 데이터 분석 시스템 입니다.
* 본인의 이름, 현재 체중, 주요 웨이트 종목(벤치프레스, 스쿼트 등)의 중량을 입력합니다.
* 시스템은 입력된 데이터를 바탕으로 오늘 수행한 '총 중량'을 계산합니다.
* 사용자가 설정한 '목표 중량'과 실제 수행 중량을 비교하여, 목표 달성까지 남은 수치를 산출합니다.
* 체중 대비 중량비를 계산하게 그에 따라 사용자의 운동 등급(헬린이, 득근자, 헬창 등)을 부여할 것입니다.
* 최종적으로는 사용자가 운동 부위를 자유롭게 선택하여 기록할 수 있도록 할 예정입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 사용자 신체 정보 및 목표 입력(이름, 체중, 목표 중량 설정)
2. 부위별 운동 중량 기록(벤치프래스, 스쿼트 등 당일 수행 무게 입력)
3. 총합 중량 및 목표 달성률 계산(전체 중량 합산 및 목표치와의 차이 분석)
4. 상대 근력 지수 기반 등급 산정(체중 대비 중량비 계산 및 등급 부여)
5. 운동 결과 리포트 및 칭호 출력(전체 기록 요약 및 조건 만족 시 특별 칭호 부여)
6. 프로그램 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  *사용자로부터 이름, 현재 체중, 목표 중량, 주요 운동(벤치프레스, 스쿼트)의 무게를 입력받는 기능을 구현했습니다.
  *이름(char), 목표 중량(int), 체중 및 운동 무게(double)을 입력받는 화면을 구현했습니다.
  *사칙연산을 활용해 '총합 중량'계산과 '목표 중량까지 남은 수치'를 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** (변수 5개 이상, 자료형 3개 이상)을 충족하면서, 실생활에서 활용 가능한 의미 있는 프로그램 시나리오 추천 요청
    * **적용 내용:** AI가 제안한 3가지 시나리오(포켓몬, 편의점, 운동) 중 **'오운완(오늘 운동 완료) 기록기'**를 최종 선택함.
      단순한 기록을 넘어, '기초 기록(1차) ➔ 근력 등급 판별(2차) ➔ 부위별 메뉴 시스템(3차) ➔ 주간 데이터 누적 관리(4차)'로 이어지는 체계적인 개발 로드맵을 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** 사용자로부터 중량 데이터를 입력받을 때, 정수와 실수를 혼용하여 입력해도 오류 없이 계산될 수 있는 안정적인 데이터 구조 설계 자문, 실수형(double)데이터 출력 시 발생하는 불필요한 소수점 자릿수를 정리하여 가독성을 높이는 방법 논의
    * **적용 내용:** 사용자가 (100)이라고 입력하든 (102.5)라고 입력하든 모두 수용할 수 있도록 중량 관련 변수를 모두 (double)로 통일함, 소수점 아래 무의미한 0들이 나열되는 것을 방지하기 위해 체중은 신체 변화를 가장 민감하게 체크해야 하는 지표이므로, 소수점 둘째 자리(%.2f)까지 정밀하게 표현하여 미세한 체중 증감까지 모니터링할 수 있도록 설계함. 헬스장에서 주로 사용하는 원판 단위(보통 2.5kg)에 맞춰 직관적으로 확인할 수 있도록 소수점 첫째 자리(.1f)로 출력하도록 설계함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** scanf로 데이터를 입력받을 때 주소 연산자(&)를 누락하여 프로그램이 비정상 종료되는 런타임 에러 발생.
     * **해결:** AI에게 질문하여 scanf는 변수의 메모리 주소에 값을 저장해야 한다는 원리를 파악하고 (&) 기호를 추가하여 해결함.
  2. **문제 2:** 실수형 데이터를 출력할 때 소수점이 너무 길게 나와 가독성이 떨어짐.
     * **해결:**  printf("%.1f", total_volume); 와 같이 서식 지정자를 수정하여 소수점 첫째 자리까지만 정교하게 출력되도록 개선함

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * 다중 조건문 (if-else if-else) 활용: 계산된 체중 대비 중량비(ratio)를 바탕으로 '헬창', '득근자',
    '헬린이'등 급을 자동으로 판별하는 로직을 구현함
  * 입력 예외 처리: 벤치프레스나 스쿼트 무게가 0 미만으로 입력되는 경우 사용자에게 오류 메시지를 출력
    하는 로직 추가
  * 메뉴 시스템 구축: switch-case문을 활용하여 상세 기록 보기, 조언 보기, 종료하기 등 사용자가 원하는 기능을 선택할 수 있는 대화형 인터페이스를 구현함
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  switch-case문을 사용하여 메뉴 번호를 선택할 때, 입력값에 없는 번호를 선택하면 자동으로 default에 있는 메시지가 출력되도록 하는 코드 구조 자문
    * **적용 내용:** break문의 중요성을 학습하고, 각 case별로 독립적인 로직을 수행하도록 배치하여 프로그램의 구조를 체계화함
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 세미콜론(;)누락으로 인한 컴파일 에러
     printf문 뒤에 세미콜론을 빠뜨려 [Error] expecter ';' before 'break'에러 발생
     * **해결:** C언어에서 문장의 끝을 명확히 정의해야 한다는 점을 다시한번 배웠고 에러 메시지를 통해 해당 줄을 즉시 찾아 수정함
       
  3. **문제 1:** 코드 블록 ({}) 괄호 불일치로 인한 문법 오류
     조건문이 중첩되면서 닫는 괄호의 개수가 맞지 않아 expected identifier or '(' before '}' token 에러가 발생함
     * **해결:** 코드를 들여쓰기하여 괄호의 짝을 눈으로 확인하는 습관을 들였고 AI의 도움을 받아 구조를 재정비하여 해결함
